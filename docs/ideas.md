# 発展アイデア

このドキュメントは、MVP の後に追加しやすい機能や企画をまとめる場所です。優先度は固定せず、小さく試せるものから実装します。

## 機能追加

- 投票機能: GitHub Issues、外部フォーム、または小さなバックエンドを使ってユーザー投票を集める。
- お気に入り保存: `localStorage` を使い、ブラウザ内にお気に入り技を保存する。
- 詳細ページ: 各必殺技に専用ページを作り、スコア内訳、タグ、関連技を表示する。
- タグ検索: `tags` をクリックして同じタグの技に絞り込めるようにする。
- 複合ランキング: 総合、かっこよさ、インパクト、象徴性を切り替えてランキング表示する。
- データ投稿フォーム: JSON 追加用のテンプレート生成フォームを用意する。
- SNS シェア: ランキング結果や個別技を共有しやすくする。
- ランダム表示: 今日の必殺技、ランダム対決、技ガチャを追加する。

## ランキング企画

- キャラ別ランキング
- 作品別ランキング
- ジャンル別ランキング
- 光線技ランキング
- 斬撃技ランキング
- キック技ランキング
- ライバルキャラの必殺技ランキング
- ボスキャラの必殺技ランキング
- 現実で真似したい技ランキング
- 名前の響きがかっこいい技ランキング
- 演出が派手なフィニッシュランキング

## データ設計の拡張

- `aliases` を追加して表記揺れ検索に対応する。
- `debut_year` を追加して年代別ランキングを作る。
- `source_type` を追加して漫画、アニメ、ゲーム、実写、試合を分ける。
- `power_type` を追加して炎、雷、光、物理、関節などで分類する。
- `related_ids` を追加して派生技や同系統の技をつなげる。
- `detail_url` を追加して詳細ページや参考リンクを扱う。
- `votes` や `user_score` を追加してユーザー評価と編集部評価を分ける。

## UI 改善

- スコア内訳によるレーダーチャート表示。
- カテゴリ別の色分け。
- ランキング上位だけの特別レイアウト。
- スクロール位置を保ったフィルタ更新。
- URL クエリに検索条件を反映し、絞り込み結果を共有可能にする。
- ダークテーマを維持しつつ、読みやすいライトテーマを追加する。

## 運用

- データ追加用の Pull Request テンプレートを作る。
- JSON 検証用の簡易スクリプトを追加する。
- GitHub Actions で JSON の構文チェックを行う。
- README にスクリーンショットと公開 URL を追加する。
- ライセンスと引用ポリシーを明記する。
